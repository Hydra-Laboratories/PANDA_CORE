## PR Review and Bug Fixes

### Summary

Ran a comprehensive PR review using specialized agents (code, tests, errors, types). Found and fixed several bugs before merge.

### Critical Bugs Fixed

1. **`driver.py` — `_GamryBackend.run_chronoamperometry` called `_run_signal` with `chrono_mode=True`**, a kwarg that does not exist in `_run_signal`'s signature. Guaranteed `TypeError` crash on every real Gamry CA measurement. Removed the spurious kwarg.

2. **`commands/measure.py` — catch block too broad**. Was catching `(sqlite3.Error, TypeError, ValueError, KeyError)` as a silent warning. `TypeError` is the documented error contract for `normalize_measurement` when it encounters an unknown result type — swallowing it would silently drop measurement data with no user notification. Narrowed to only catch `sqlite3.Error` (genuine I/O failure), changed to `logger.error`, let all other exceptions propagate.

3. **`driver.py` — `_series_from_mapping` silently returned empty tuples or filled with `0.0`** when hardware returned no data or keys didn't match. This produced `is_valid=False` result objects with no exception or log. Changed to raise `PotentiostatMeasurementError` for all missing-data cases.

### Important Fixes

4. **`driver.py` — `_GamryBackend.disconnect()` empty `except: pass`** — replaced with `logger.warning`.

5. **`driver.py` — `_run_signal` finally block empty `except: pass`** — failing to turn off the Gamry cell after a measurement is a hardware safety concern; replaced with `logger.error`.

6. **`driver.py` — `_run_hardpotato_experiment` used `getattr(experiment, "data", None)`** silently returning `None` into `_series_from_mapping`. Added explicit guard for `class_name` not existing on `hp.potentiostat`, and changed to direct attribute access that raises if `data` is absent.

7. **`commands/measure.py` — mutable default argument `method_kwargs: Dict[str, Any] = {}`** — changed to `None` with guard at top of function.

8. **`measurements.py` — `**raw_result.metadata` spread came after explicit keys** like `"vendor"` and `"technique"`, so any matching key in `metadata` would silently overwrite explicit values. Moved spread before explicit keys so explicit values always win.
