# 2026-04-08 — Package `instruments/registry.yaml`

## Summary

Fixed a packaging bug where `src/instruments/registry.yaml` existed in the repository but was omitted from built artifacts, causing `instruments/registry.py` to fail at runtime after install.

## What Was Done

- Added `instruments = ["registry.yaml"]` to `[tool.setuptools.package-data]` in `pyproject.toml`
- Added `tests/instruments/test_registry_packaging.py` to build a wheel and sdist from an isolated copy of the repository and assert both artifacts contain the registry YAML

## Root Cause

- The setuptools package data configuration only included JSON assets for `gantry.gantry_driver`
- Because `instruments/registry.yaml` was not declared as package data, setuptools copied `registry.py` but not the sibling YAML file into the wheel or sdist
- Downstream installs from Git then produced an installed `instruments` package without `registry.yaml`, which broke the existing runtime lookup in `instruments/registry.py`

## Verification

- `pytest tests/instruments/test_registry_packaging.py` -> passed
- `pytest tests/instruments/test_registry.py` -> passed
- Temporary-copy build verification using `setuptools.build_meta` confirmed:
  - `wheel_has_registry=True`
  - `sdist_has_registry=True`
  - `installed_has_registry=True`
