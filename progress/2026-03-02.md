## Task: Reconfigure Gantry Axes to Positive Coordinate Space (Planning)

### Work Completed
- Reviewed current gantry stack and confirmed there is no existing user/machine translation boundary.
- Verified `src/gantry/gantry.py` currently passes coordinates directly to `Mill.safe_move()` and returns raw mill coordinates/status to callers.
- Reviewed config/schema/loader flow and confirmed gantry YAML is still negative-space in `configs/gantry/*.yaml`.
- Reviewed command behavior and confirmed `src/protocol_engine/commands/scan.py` currently applies `well.z + measurement_height`.
- Reviewed deck/board/validation paths and confirmed they currently operate in a unified coordinate space (no sign conversion layer there).
- Reviewed high-impact test files and confirmed widespread negative-space assumptions in gantry, board, deck, validation, and protocol tests.

### Key Findings
- The translation-layer approach in `Gantry` is sound and keeps the mill driver internals unchanged.
- Sequencing matters: enabling translation in `Gantry` before flipping deck/board/test coordinates will temporarily invert behavior and break many tests.
- `total_z_height` introduction is straightforward in schema/loader/domain, but needs a clear source-of-truth location (`cnc.total_z_height` vs `working_volume.z_max` only).
- `scan` measurement-height sign must be updated for positive-down Z semantics.

### Risks Identified
- User-facing setup scripts with manual jog semantics may need sign updates once `Gantry.move_to()` becomes positive-space.
- Status-string translation must preserve GRBL formatting and handle both `MPos` and `WPos` robustly.
- Labware `height` support introduces a loader API change; this should remain backward compatible.

### Next Planned Step
- Finalize implementation order and acceptance criteria with user confirmation before coding.

## Implementation Completed

### Scope Delivered
- Implemented a hard-breaking migration to positive user-facing gantry coordinates while keeping GRBL/mill internals in machine space.
- Added a dedicated coordinate translation boundary in `Gantry` and new translation utilities.
- Migrated gantry schema/config/domain to include `cnc.total_z_height` (required, `> 0`, and `>= working_volume.z_max`).
- Updated deck loader/schema for optional labware `height` support with computed z (`z = total_z_height - height`).
- Updated protocol scan Z offset behavior for positive-down semantics (`well.z - measurement_height`).
- Updated user-facing setup/config/docs to match positive coordinate conventions.

### Key Code Changes
- **New translation module**
  - `src/gantry/coordinate_translator.py`
  - Added tuple/object translation helpers and GRBL status-string coordinate translation (`MPos`/`WPos`).
- **Gantry translation integration**
  - `src/gantry/gantry.py`
  - `move_to()` now translates user -> machine coordinates before calling `Mill.safe_move()`.
  - `get_coordinates()` now translates machine -> user coordinates.
  - `get_status()` now translates displayed coordinates.
  - Added `total_z_height` property passthrough.
- **Gantry config/schema/loader**
  - `src/gantry/gantry_config.py`: added `total_z_height` to `GantryConfig`.
  - `src/gantry/yaml_schema.py`: added `cnc.total_z_height` validation rules.
  - `src/gantry/loader.py`: now loads `total_z_height`.
  - Updated `configs/gantry/genmitsu_3018_PROver_v2.yaml` and `configs/gantry/genmitsu_3018_PRO_Desktop.yaml` to positive bounds + `total_z_height`.
- **Deck height helper**
  - `src/deck/yaml_schema.py`: optional `height` on well plate/vial entries, optional point `z`.
  - `src/deck/loader.py`: `load_deck_from_yaml(path, total_z_height=None)` and safe variant support computed z from height.
  - `src/protocol_engine/setup.py` and `setup/validate_setup.py`: pass gantry `total_z_height` into deck loading.
- **Protocol behavior**
  - `src/protocol_engine/commands/scan.py`: measurement target z changed to `well.z - measurement_height`.
- **Setup script**
  - `setup/hello_world.py`: updated config path folder names, positive-space jog semantics, and movement clamping to working-volume bounds.
- **Mock + docs/docstrings**
  - `src/gantry/gantry_driver/mock.py`: positive mock working volume and safe floor height.
  - `src/validation/bounds.py`, `src/board/board.py`: clarified positive-space semantics in docstrings.
