# 2026-03-09 — WPos Enforcement in Mill Driver

## Work Done

### 1. Enforced WPos mode at Mill driver level
- `Mill._enforce_wpos_mode()` runs on connect: sets `$10=0` (WPos reporting) and `G90` (absolute positioning)
- This ensures `current_coordinates()` always receives WPos directly from GRBL
- Updated `_configuration.json` default to `$10=0`

### 2. Fixed MPos→WPos conversion in `Mill.current_coordinates()`
- Replaced the empty `pass` on the MPos branch with actual WCO-based conversion
- `WPos = MPos - WCO` when GRBL reports MPos (fallback for non-standard configs)
- WCO is cached from GRBL status reports and updated on every coordinate query

### 3. Added WCO caching to Mill
- `Mill._wco` field caches the Work Coordinate Offset
- `Mill._seed_wco()` polls GRBL status up to 15 times to capture WCO after connect
- `Mill._query_work_coordinate_offset()` returns cached WCO (seeds if needed)
- WCO is also updated opportunistically from every status report in `current_coordinates()`

### 4. Added `Mill.machine_coordinates()`
- Returns raw MPos computed as `WPos + WCO`
- Needed by hardware test and for debugging

### 5. Updated MockSerialToMill
- Mock now reports WPos format (was MPos) with WCO included in status strings
- This makes tests accurately reflect the enforced WPos mode

### 6. Fixed Python 3.9 compatibility
- Changed `float | None` type hint to `Optional[float]` in `Gantry.total_z_height`

### 7. Added unit tests
- `tests/gantry/driver/test_wpos_enforcement.py` — 8 tests covering:
  - WPos returned directly when GRBL reports WPos
  - MPos correctly converted to WPos using WCO
  - WCO caching from status reports
  - `machine_coordinates()` returns MPos
  - `safe_move()` uses WPos internally

## Coordinate Flow (after fix)
```
GRBL ($10=0) → WPos → Mill.current_coordinates() → WPos
                                   ↓
                     Mill.safe_move() internal logic (all WPos)
                                   ↓
                     G-code commands (G01 X... Y... Z...) in WPos
                                   ↓
                     Gantry.get_coordinates() → to_user_coordinates() → positive user space
                     Gantry.move_to() ← to_machine_coordinates() ← positive user space
```
