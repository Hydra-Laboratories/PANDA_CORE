## DataReader + UV-Vis Analysis Module

### Work Done
Added read-side query layer and UV-Vis specific analysis helpers.

**`data/data_reader.py`** (new) — Read-only query interface
- `DataReader` class: accepts `db_path` or existing `sqlite3.Connection`
- `CampaignRecord`, `ExperimentRecord`, `LabwareRecord` frozen dataclasses
- `get_campaign()`, `list_campaigns()` — campaign metadata + config snapshots
- `get_experiments()` — filter by campaign, labware_name, well_id
- `get_labware()` — filter by campaign, labware_key
- `get_measurements()` — generic raw rows from any measurement table (with SQL injection guard)
- `get_measurements_by_campaign()` — joins experiments to get well_id/labware_name with measurements
- Context manager support

**`data/analysis/uvvis.py`** (new) — UV-Vis specific read + analysis
- `UVVisRecord` frozen dataclass (unpacked wavelengths, intensities, metadata)
- `unpack_spectrum()` — BLOB → tuple[float, ...] deserialization
- `load_uvvis_by_experiment()` / `load_uvvis_by_campaign()` — load spectra from DB via DataReader
- `peak_wavelength()` — find (wavelength, intensity) at max intensity
- `absorbance()` — compute A = -log10((sample - dark) / (reference - dark))
- `slice_wavelength_range()` — filter spectrum to [wl_min, wl_max]

### Design
- **DataReader** is generic (campaigns, experiments, labware, any measurement table)
- **analysis/uvvis.py** is instrument-specific (BLOB format, spectral operations)
- DataReader accepts a `connection` param for sharing with DataStore in tests
- Valid measurement tables are whitelisted to prevent SQL injection
