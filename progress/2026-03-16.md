# 2026-03-16: ASMI Gantry Extensions for ASMI_new Integration

## Branch: `feature/asmi-gantry-extensions`

## Work Done

### Gantry Extensions (`src/gantry/gantry.py`)
Added single-axis movement methods and utilities required by ASMI_new's step-by-step indentation workflow:

- **`move_z(z, feed, wait)`** - Z-only absolute movement with optional idle wait
- **`move_xy(x, y, feed)`** - XY-only absolute movement (Z unchanged)
- **`get_position()`** - Returns (x, y, z) tuple, raises on error (unlike `get_coordinates()` which returns defaults)
- **`_wait_for_idle(timeout)`** - Polls GRBL status until Idle state
- **`unlock()`** - Sends GRBL $X to clear alarm/lock state

### ASMI Instrument Extensions (`src/instruments/asmi/driver.py`)
Added convenience methods matching ASMI_new's ForceSensor API:

- **`get_force_reading()`** - Single reading returning float (wraps `measure(n_samples=1)`)
- **`get_baseline_force(samples)`** - Returns (mean, std) tuple (wraps `measure(n_samples=N)`)
- **`is_connected()`** - Quick connection check without taking a measurement

### ASMI Instrument Added to Branch
The ASMI instrument module (`src/instruments/asmi/`) was brought into this branch from `uvvis-sql`. Files: `__init__.py`, `driver.py`, `exceptions.py`, `mock.py`, `models.py`.

## Context
These extensions support the ASMI_new refactoring (`ASMI_new` branch: `refactor/use-panda-core`) which replaces its standalone CNCController and ForceSensor with PANDA_CORE's Gantry and ASMI instrument.
