# 2026-04-16 — CubXL First-Time Connection (In Progress)

## Goal

Connect the CubXL (Genmitsu 3018 PRO) to Mac for the first time.

## Status: Blocked — GRBL firmware not yet flashed

### What we confirmed

1. **USB serial driver is working.** macOS is NOT blocking the driver. Device shows up at `/dev/tty.usbserial-130` (and `/dev/cu.usbserial-130`).
2. **`setup/test_connection.py` runs but fails** — the controller board doesn't respond to GRBL commands. Empty responses (`Raw response: []`) at both 115200 and 9600 baud across 3 scan attempts.
3. **Root cause: GRBL firmware has not been flashed onto the board yet.** Alex downloaded a flashing tool/firmware but suspects macOS Gatekeeper blocked the installer or binary.

### Next steps to resume

1. **Unblock the firmware flashing tool on macOS.** Likely need to:
   - Check System Settings > Privacy & Security for a "blocked" notification and click "Allow Anyway"
   - Or right-click the app/binary and choose "Open" to bypass Gatekeeper
   - Alternative: install `avrdude` via Homebrew (`brew install avrdude`) which won't be blocked, and flash GRBL hex file from the command line
2. **Flash GRBL firmware** onto the Genmitsu 3018 PRO controller (ATmega328P). Typical command:
   ```bash
   avrdude -p atmega328p -c arduino -P /dev/tty.usbserial-130 -b 115200 -U flash:w:grbl_v1.1h.hex:i
   ```
   (Exact hex file and baud rate depend on what was downloaded.)
3. **Re-run connection test:**
   ```bash
   python setup/test_connection.py --gantry cub_xl --port /dev/tty.usbserial-130
   ```
4. If connection succeeds, proceed to homing and GRBL settings configuration per `configs/gantry/cub_xl.yaml`.
